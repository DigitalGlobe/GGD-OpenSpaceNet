std::unique_ptr<std::string> dg::deepcore::imagery::TileJson::_template

OPTIONAL. Default: null. Contains a mustache template to be used to format data from grids for interaction. See https://github.com/mapbox/utfgrid-spec/tree/master/1.2 for the interactivity specification.

std::unique_ptr<std::string> dg::deepcore::imagery::TileJson::attribution

OPTIONAL. Default: null. Contains an attribution to be displayed when the map is shown to a user. Implementations MAY decide to treat this as HTML or literal text. For security reasons, make absolutely sure that this field can't be abused as a vector for XSS or beacon tracking.

cv::Rect2d dg::deepcore::imagery::TileJson::bounds = { -180, -MERCATOR_90_LAT, 360, MERCATOR_90_LAT * 2 }

OPTIONAL. Default: { -180, -85.0511287798066, 360, 170.10225756 } The maximum extent of available map tiles. Bounds MUST define an area covered by all zoom levels. The bounds are represented in WGS:84 latitude and longitude values.

std::unique_ptr<std::string> dg::deepcore::imagery::TileJson::description

OPTIONAL. Default: null. A text description of the tileset. The description can contain any legal character. Implementations SHOULD NOT interpret the description as HTML.

std::string dg::deepcore::imagery::TileJson::format = "jpg"

NON-STANDARD, Image format, i.e. "jpg"

std::vector<std::string> dg::deepcore::imagery::TileJson::grids

OPTIONAL. Default: []. An array of interactivity endpoints. {z}, {x} and {y} are replaced with the corresponding integers. If multiple endpoints are specified, clients may use any combination of endpoints. All endpoints MUST return the same content for the same URL. If the array doesn't contain any entries, interactivity is not supported for this tileset. See https://github.com/mapbox/utfgrid-spec/tree/master/1.2 for the interactivity specification.

std::unique_ptr<std::string> dg::deepcore::imagery::TileJson::legend

OPTIONAL. Default: null. Contains a legend to be displayed with the map. Implementations MAY decide to treat this as HTML or literal text. For security reasons, make absolutely sure that this field can't be abused as a vector for XSS or beacon tracking.

int dg::deepcore::imagery::TileJson::maxzoom = 22

OPTIONAL. Default: 22. >= 0, <= 22. An integer specifying the maximum zoom level. MUST be >= minzoom.

int dg::deepcore::imagery::TileJson::minzoom = 0

OPTIONAL. Default: 0. >= 0, <= 22. An integer specifying the minimum zoom level.

std::unique_ptr<std::string> dg::deepcore::imagery::TileJson::name

OPTIONAL. Default: null. A name describing the tileset. The name can contain any legal character. Implementations SHOULD NOT interpret the name as HTML.

std::string dg::deepcore::imagery::TileJson::scheme = "xyz"

OPTIONAL. Default: "xyz". Either "xyz" or "tms". Influences the y direction of the tile coordinates. The global-mercator (aka Spherical Mercator) profile is assumed.

std::string dg::deepcore::imagery::TileJson::tilejson

REQUIRED. A semver.org style version number. Describes the version of the TileJSON spec that is implemented by this JSON object.

std::vector<std::string> dg::deepcore::imagery::TileJson::tiles

REQUIRED. An array of tile endpoints. {z}, {x} and {y} are replaced with the corresponding integers. If multiple endpoints are specified, clients may use any combination of endpoints. All endpoints MUST return the same content for the same URL. The array MUST contain at least one endpoint.

std::string dg::deepcore::imagery::TileJson::version = "1.0.0"

OPTIONAL. Default: "1.0.0". A semver.org style version number. When changes across tiles are introduced, the minor version MUST change. This may lead to cut off labels. Therefore, implementors can decide to clean their cache when the minor version changes. Changes to the patch level MUST only have changes to tiles that are contained within one tile. When tiles change significantly, the major version MUST be increased. Implementations MUST NOT use tiles with different major versions.
