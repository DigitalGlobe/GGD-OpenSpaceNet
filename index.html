Deep Learning SDK for Geospatial Data

Welcome to DeepCore

DeepCore is a machine learning abstraction framework and utility toolkit, written in modern C++(C++11) by DigitalGlobe. It allows a user to download, perform object detection, and manipulate geospatial vector files. DeepCore is intended to be a machine learning framework agnostic toolkit, allowing for a simple, clean, consistent programmatic interface. It also provides easy access to the DigitalGlobe imagery archive. As new machine learning techniques and frameworks emerge, they can easily be integrated into DeepCore. This allows developers using DeepCore to easily extend their applications with the latest technology, without having to worry about the complexities of each framework or technique.

DeepCore's machine learning features can also be accelerated by the use of Nvidia Graphics Processing Units (GPUs) using CUDA technology. By enabling GPU mode, the process of object detection becomes very quick, allowing for faster and more efficient processing of large geographic areas. The use of GPUs to accelerate machine learning and object detection processes is highly recommended.

DeepCore is split into libraries that are grouped by function. The core libraries are:

  • Utility - contains toolkit type functions that are very general and don’t fit into any of the other categories. This includes logging via Google glog, zip file reading and writing, basic math, and other basic functions.
  • Network - contains functions and classes for communication across networks, including HTTP and JSON-RPC requests
  • Vector - contains functions and classes for writing vector features out to disk or network sources. Supported formats are:
    • Shapefile
    • KML
    • PostGRES/PostGIS
    • ElasticSearch
    • GeoJSON
  • Imagery - contains functions and classes for reading imagery from a variety of sources. Using GDAL and OpenCV, most image formats can be read in for processing.
  • Classification - contains the functions and classes necessary to do classification and object detection with a supported model. Currently, only Caffe models are supported, but additional frameworks such as Microsoft’s CNTK and Google Tensorflow are coming online soon.

For more information, please see the DeepCore API documentation found here.

Sample Application - gbdxm

The GBDXM packaging tool is a DeepCore based executable that enables the user to package a machine learning model for use with other DeepCore based tools. It takes in a supported model format as an input, and packages the model files into a single easy-to-use, easy-to-share container. The DeepCore framework can then use that model package to perform machine learning and object detection tasks. Currently, only Caffe models are supported, but other model formats are forthcoming as additional machine learning frameworks are incorporated into DeepCore. The GBDXM application can be downloaded via link.

Sample Application - OpenSpaceNet

OpenSpaceNet is a DeepCore-based sample application that uses DigitalGlobe satellite imagery as an input source for the object detection. Using DigitalGlobe image services, such as the DigitalGlobe Maps API or the My DigitalGlobe imagery service, the user can access current satellite imagery for use in machine learning. OpenSpaceNet allows the user to select an area for detection, specify a model to use during detection, and get the results back in a geospatial format suitable for use in a GIS application or service. The OpenSpaceNet application can be downloaded via link or by clicking the button on the right.

How-to and getting started

To get started using DeepCore, download and uncompress the dependency package into the /opt directory. Then download and uncompress the DeepCore tarball. This will place the libraries and headers on your local machine. To create tools and applications that use DeepCore, just make sure to include the headers directory and add the lib directory to your linker path or if using CMake, just add the share directory to your CMAKE_MODULE_PATH and use 
                
find_package(DeepCore REQUIRED)
if (DeepCore_FOUND)
    include_directories(${DEEPCORE_INCLUDE_DIRS})
    target_link_libraries(OpenSpaceNet ${DEEPCORE_LIBRARIES})
    list(APPEND CMAKE_MODULE_PATH ${DEEPCORE_CMAKE_DIR})
endif()

Dependencies

To be able to build applications with DeepCore, you will need the third party dependencies bundle. Please download the bundle corresponding to your local CUDA installation. The tarball bundles need to be uncompressed into the /opt directory.
  • CUDA 7.5 linked dependencies
  • CUDA 8 linked dependencies

For more information on setting up your DeepCore environment, see our Building page.

DeepCore distributions

The DeepCore binaries and headers come as a tarball that can be extracted to your local drive. They are also segmented by CUDA version dependencies.
  • DeepCore (CUDA 7.5)
  • DeepCore (CUDA 8.0)
